RESOURCE TAGS AND BASIC POOL

Six basic resources:
- bone: basic, materialTags ['bone']
- hide: basic, materialTags ['hide']
- organ: basic, materialTags ['organ']
- sinew: basic, materialTags ['organ'] because sinew is classed as organ in this ruleset
- scrap: basic, materialTags ['scrap']; not valid for the Bone/Hide/Organ innovation slots unless a rule says so
- loveJuice: basic but slightly rarer, materialTags ['organ']; special intimacy protection use

Required code shape:
function resource(id, name, type, description, creatureId, options = {}) {
  return {
    id,
    name,
    type,
    description,
    ...(creatureId ? { creatureId } : {}),
    materialTags: options.materialTags || [],
    basicResource: Boolean(options.basicResource),
    genericDropWeight: options.genericDropWeight ?? 0,
    specialUse: options.specialUse || null
  };
}

Explicit genericResourceIds:
export const genericResourceIds = ['bone', 'hide', 'organ', 'sinew', 'scrap', 'loveJuice'];

Suggested generic drop weights:
- bone: 60
- hide: 60
- organ: 60
- sinew: 60
- scrap: 45
- loveJuice: 12

Rule for multi-tag resources:
A resource can have one or two body material tags. If it has two, one copy can pay for only one selected slot at payment time.

Name heuristic fallback for future resources:
- contains hide, leather, pelt, mane, fur, feather, scale, wing, cloak, plate-like skin => hide
- contains bone, fang, tooth, claw, horn, shell, shard, plate, crown, thorn, stem, marrow => bone
- contains organ, heart, eye, gland, lung, sac, ichor, blood, sinew, tongue, thread, seed, core => organ
- contains token, scrap, fragment, blade, mirror, glass, metal, shard, due, verdict => scrap, unless a stronger body tag is obvious

Do not rely only on heuristics for existing resources. Existing resources should receive explicit materialTags from the map above.
