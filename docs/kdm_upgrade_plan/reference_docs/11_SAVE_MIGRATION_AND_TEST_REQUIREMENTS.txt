SAVE MIGRATION AND TEST REQUIREMENTS

Migration requirements:
- Ensure old saves without loveJuice load safely.
- Ensure old saves with claw/fur/horn/ichor/monsterTooth/strangeEye keep those resources, but they no longer count as basic generic drops.
- Add materialTags at runtime for old resources if saved resource objects exist anywhere.
- Maintain memories and settlementMemory as synced aliases until old UI code is fully removed.
- Migrate builtMemoryInnovations into innovationDeckState.builtInnovationIds where appropriate.
- Do not crash on unknown/legacy innovation ids.
- Pending newborn/intimacy state must survive migration.
- Old innovation history should still display, even if paidResources was an array rather than slot mapping.
