KDM / Lantern Deckbuilder
GEAR OVERHAUL REPO MAP
Recommended repo path: docs/KDM_GEAR_OVERHAUL_REPO_MAP.txt

Purpose
-------
This is a planning and implementation map for replacing the old imported equipment/card catalogue with a smaller original quarry-location gear system.

It is based on the current repo structure:
- src/data/quarries.js controls quarry IDs, quarry tiers, design tags, loot/resource family, and building unlocks.
- src/data/resources.js controls resource IDs and names.
- src/data/equipment.js currently builds equipment from src/data/overhaul/gearRegistry.js.
- src/data/overhaul/gearRegistry.js currently imports gear_master_overhaul.json and all_deck_cards.json.
- src/data/cards.js currently merges overhaulCards from src/data/overhaul/cardRegistry.js, weapon mastery cards, starter cards, monster rewards, and fighting art cards.
- src/data/weaponProficiency.js controls weapon type list, proficiency thresholds, proficiency passive text, and mastery card IDs.
- src/data/gearMetadata.js currently has explicit old gear IDs and weapon type metadata.
- src/data/events.js contains locked event choices based mainly on weaponType, gearTag, traits, innovations, and some generic resources.
- src/game/eventRequirementLogic.js supports locked choice requirements including partyHasWeaponType, partyHasGearTag, partyHasGearId, traits, fighting arts, conditions, innovations, and monster bane knowledge.

Important design rule
---------------------
Keep this original. Do not copy Kingdom Death names, prose, cards, or mechanics. The inspiration is only:
- dark settlement survival;
- monster/quarry-specific crafting;
- hunt/showdown flow;
- resources from defeated creatures;
- scary readable equipment.

Do not edit:
- src/components/Card.jsx flip structure;
- card flip CSS unless fixing a pure layout bug;
- dist/;
- node_modules/.

The new card text must stay short and player-friendly.
Front card text should look like:
- "Deal 3 damage. Apply Poison 1."
- "Gain 4 Block. Your next attack deals +2 damage."
- "Aura — All Block cards give +1 more Block this fight."

Dense mechanics should go in comments, Details/Advanced text, or card-back tactical preview, not the front text.


A. CURRENT REPO FACTS THAT MATTER
=================================

A1. Crafting costs
------------------
Crafting cost is a plain object of resourceId to amount:

Example:
cost: {
  bone: 1,
  paleLionHide: 1
}

The existing crafting logic:
- checks missing resources by resourceId;
- formats cost names through src/data/resources.js;
- deducts exact resource IDs from settlement stash.

Therefore every new gear item needs:
- id
- name
- buildingId / locationId / craftingLocationId
- quarryId if quarry gear
- itemType / loadoutCategory / slot / bodySlot where relevant
- weaponType if it is a weapon
- cost object with valid resource IDs
- cardPackage array of valid card IDs
- clear player-facing description/passiveText.

A2. Quarry loot/resource rule
-----------------------------
The current quarries build loot by quarry:
- Level 1 loot uses creature/common parts.
- Level 2 loot uses common parts plus rare/strange parts.
- Level 3 loot uses common + rare/strange + level3Rare parts.

Use that logic for cost gating:
- early/basic gear should use common creature parts and basic resources;
- stronger gear should use rare/strange parts;
- apex/signature gear should use level3Rare parts.

A3. Current loadout direction
-----------------------------
Current loadout rules already support:
- max 9 equipment per survivor;
- max 2 weapon items;
- duplicate weapon names allowed;
- armour body slots;
- jewellery slot;
- tools/utilities within the 9 item cap.

The gear overhaul should respect that rather than inventing a separate loadout model.


B. RESOURCE FAMILIES AND COST PALETTES
======================================

General resource cost principles
--------------------------------
Small / fast weapons:
- Dagger, Katar, Fist & Tooth
- Cost 1 common creature hard part, or 1 generic Bone + 1 light quarry part.
- Example pattern: { creatureToothOrClaw: 1 } or { bone: 1, quarryHideOrSinew: 1 }

Medium weapons:
- Sword, Spear, Bow, Whip, Axe, Shield
- Cost 1-2 common creature parts plus 0-1 basic resource.
- Example pattern: { quarryHardPart: 1, hide: 1 }

Technical weapons:
- Katana
- Cost a clean precise part plus sinew/bone.
- Example pattern: { quarryPrecisePart: 1, sinew: 1 }
- Stronger katana versions may add one rare/strange part.

Heavy weapons:
- Grand weapons
- Cost 2+ common parts and usually 1 rare/strange part.
- Late/apex grand weapons may require a level3Rare part.
- Example pattern: { quarryBoneOrHorn: 2, quarryRare: 1 }

Armour:
- Each armour set should use the defensive hide/scale/shell/chitin resource.
- Individual pieces should be cheaper than weapons.
- Complete set identity comes from passives/cards, not huge text.
- Example light piece: { quarryHide: 1 }
- Example chest/strong piece: { quarryHide: 2, hide: 1 }
- Example late piece: { quarryPlate: 1, quarryRare: 1 }

Instruments:
- Use organs, horns, lungs, pipes, rhythm/vibration resources, or rare voice/light/memory resources.
- Cost should be 1 thematic resource + 1 basic, or 1 rare for strong instruments.

Tools:
- Tools should be specific and cheap-to-medium.
- Use odd parts: eye, gland, tongue, pipe, charm, sac, thread, ash, scrap, hide.
- A powerful tool can use a rare/strange part.

C. SUPPORT / BASIC LOCATIONS
============================

C1. Bone Smith
--------------
Do NOT give Bone Smith a full quarry-style suite.
Bone Smith should be basic weapons only.

Suggested role:
- starter/foundation weapon upgrades;
- teaches weapon classes;
- low cost;
- few special rules;
- uses generic bone/hide/sinew/scrap.

Allowed:
- basic sword
- basic dagger
- basic axe
- basic spear
- basic shield
- basic bow
- basic grand weapon
- basic fist/tooth wraps
- maybe basic katars/katana if needed for weapon class access

Not allowed:
- full armour set
- full instrument line
- quarry identity tools
- late-game effects

C2. Skinnery
------------
Do NOT give Skinnery a full quarry-style suite.
Skinnery should be basic hide armour and survival gear only.

Allowed:
- basic armour pieces
- hide wraps
- survival cloak/boots/hood
- simple passive block gear
- basic tool like grease/salve if already needed

Not allowed:
- one of each weapon
- full instrument line
- quarry identity weapons

C3. Organ Grinder
-----------------
Do NOT give Organ Grinder a full quarry-style suite.
Organ Grinder should just be tools/support/instruments if desired.

Allowed:
- tools
- organs/charm utility
- healing/debuff/buff items
- maybe one or two instruments

Not allowed:
- one of every weapon
- armour set
- full quarry identity suite

C4. Lantern Hearth / Lantern Hoard
----------------------------------
Out of scope for the quarry-unlocked gear catalogue.
Keep as settlement/support only unless there is a later separate Lantern gear pass.


D. WEAPON PROFICIENCY REWORK
============================

Current weapon types in repo include:
sword, axe, dagger, spear, bow, club, hammer, grandWeapon, katar, fistAndTooth, shield, whip, scythe, katana, strangeWeapon.

For the new weapon plan, keep current valid type IDs for compatibility but update the passive rewards so they match the intended identity.

Recommended proficiency passives
--------------------------------

sword:
- Level 1: First Sword attack each turn deals +1 damage.
- Level 2: After you play a Sword attack, your next Block card this turn gives +1 Block.
- Mastery: Once per fight, after a Sword card wounds, draw 1 card.
Identity: reliable, flexible, mid damage, mid cost.

axe:
- Level 1: Axe attacks deal double damage to monster Block.
- Level 2: When an Axe removes all monster Block, your next attack deals +2 damage.
- Mastery: First Axe attack each fight also applies Vulnerable 1.
Identity: armour/block shredder, not simply generic high damage.

dagger:
- Level 1: First Dagger card each turn costs 0.
- Level 2: Dagger attacks apply +1 extra Bleed/Poison/Burn if they already apply a DoT.
- Mastery: After you play two Dagger cards in one turn, draw 1 card.
Identity: fast cheap DoT stacking, low direct damage.

spear:
- Level 1: First Spear attack each turn also gives +1 Block.
- Level 2: If you have Block, Spear attacks deal +1 damage.
- Mastery: Once per fight, after a Spear attack, give another survivor 3 Block.
Identity: reach simulated through damage plus small block.

bow:
- Level 1: First Bow attack each turn makes the quarry less likely to target this survivor.
- Level 2: Bow attacks gain +2 on weak-point/harvest tests.
- Mastery: Once per fight, after a Bow attack wounds, clarify the next quarry tell.
Identity: lower damage, safer, precise weak-point control, little DoT.

whip:
- Level 1: Whip attacks that damage the quarry apply Snare 1.
- Level 2: Whip cards make the quarry more likely to target this survivor until next quarry turn.
- Mastery: Once per fight, when this survivor is targeted, gain 5 Block before damage.
Identity: snare/control and drawing attention deliberately.

grandWeapon:
- Level 1: Grand Weapon attacks deal +2 damage.
- Level 2: First Grand Weapon card each fight costs 1 less.
- Mastery: Once per fight, after a Grand Weapon attack wounds, gain 4 Block.
Identity: expensive huge damage, but helps offset openness slightly at mastery.

katar:
- Level 1: If two Katars are equipped, Katar attacks cost 1 less, minimum 0.
- Level 2: After a Katar card damages the quarry, your next Katar card this turn deals +1 damage.
- Mastery: Once per turn, after playing two Katar cards, draw 1 card.
Identity: dual-wield speed; standard otherwise.

fistAndTooth:
- Level 1: First Fist & Tooth card each turn costs 0.
- Level 2: Fist & Tooth Aura cards last 1 extra turn if they target only this survivor.
- Mastery: Once per fight, when wounded, draw 1 and gain 3 Block.
Identity: low cost, low damage, more aura/self-buff support.

shield:
- Level 1: First Shield card each turn gives +1 Block.
- Level 2: When Block prevents damage, your next Shield attack deals +2 damage.
- Mastery: Once per fight, deal damage equal to half your Block, up to 20.
Identity: build block, then convert into counter damage.

katana:
- Level 1: The first Katana setup card each fight costs 0.
- Level 2: Katana setup bonuses do not expire until your next Katana attack.
- Mastery: Once per fight, double the stored bonus on your next Katana strike.
Identity: patience, stored power, single big strike.

strangeWeapon:
- Keep but do not use as a normal weapon class unless events still require it.
- Suggested: weird panic/memory gear only.
- Level 1: First Strange card each fight may remove 1 Panic or draw 1.
- Level 2: When you gain Panic from your own Strange card, gain 1 Survival.
- Mastery: Once per fight, convert 1 Panic into +4 damage or +4 Block.
Identity: dangerous occult utility.

club / hammer / scythe:
- Current repo supports these, but they are not in the current planned weapon class list.
- Either keep them as legacy/rare/special weapons or map:
  - club/hammer -> axe or grandWeapon style if deleting;
  - scythe -> strangeWeapon or grandWeapon if deleting.
- Do not leave active events requiring a deleted weapon type.

G. OLD EQUIPMENT REMOVAL PLAN
=============================

Current old equipment pipeline:
- src/data/overhaul/gearRegistry.js imports gear_master_overhaul.json and all_deck_cards.json.
- src/data/overhaul/cardRegistry.js imports all_deck_cards.json.
- src/data/equipment.js imports gearRegistry.
- src/data/cards.js imports overhaulCards.

Clean replacement options:

Option 1: Replace overhaul registry output with curated new data.
- Add src/data/gear/gearCatalog.js
- Add src/data/gear/gearCards.js
- Change src/data/equipment.js to use gearCatalog instead of gearRegistry.
- Change src/data/cards.js to import gearCards instead of overhaulCards.
- Keep old files in repo but no longer active.
- Lowest risk for quick development.

Option 2: Rewrite existing gearRegistry/cardRegistry.
- Replace gear_master_overhaul.json and all_deck_cards.json.
- Higher risk because existing assertions expect hundreds of rows.
- Not recommended.

Recommended:
Use Option 1.
Do not delete old JSON files in the first patch. Disconnect them from active imports and leave them as archived raw source until the new curated catalogue is stable.
