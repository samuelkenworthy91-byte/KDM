Purpose
-------
This file is a build guide for Codex, Gemini, Claude Code, or another AI coding agent.
It supplements the existing gear overhaul repo map.

It defines:
- the final signature item plan;
- exact card text for every signature item card;
- exact passive text for every signature item evolution level;
- the consumable card catalogue;
- the Lost pile rule for one-use consumables;
- implementation rules so the new gear works with the current deckbuilder structure.

Important design rule
---------------------
Keep all content original. Do not copy Kingdom Death names, prose, cards, or mechanics.
The inspiration is only:
- dark settlement survival;
- monster/quarry-specific crafting;
- hunt/showdown flow;
- resources from defeated quarry creatures;
- scary readable equipment.

Keep player-facing card text short and clear.
Dense implementation details can go into code comments, card back/details text, or tactical preview.

Major naming correction
-----------------------
The Crystal Forge grand weapon is named:
Dragon Slayer: Grand Cleaver

Do not use the old working name "Ember-Crown Brand".

B. LOST PILE RULE FOR CONSUMABLES
=================================

Consumable cards are powerful, craftable, one-use cards.

Card tags:
- consumable
- singleUse
- lost

When a consumable card is played:
1. Resolve its effect.
2. Move the card to a combat-only Lost pile instead of discard.
3. Mark that card instance as consumed for this encounter.

Lost cards can still be recovered during the same fight if a card explicitly says it can recover a card from Lost.
Example: the Wailing Antelope signature weapon card Second Stomach can return a consumable card from Lost or discard to hand.

Important:
A card recovered from Lost is still marked as consumed for this encounter.
At the end of the encounter, every card instance that was marked consumed must be deleted from the run deck and from any owning consumable gear/inventory instance, no matter where it currently is.

This means:
- a consumable can be used, moved to Lost, recovered, and used again in the same fight by special effects;
- it still disappears permanently after the encounter;
- the settlement can craft it again later.

Suggested state:
- combat.lostPile: card instance IDs currently in Lost;
- combat.consumedCardInstanceIds: every consumable card instance used this encounter;
- endEncounterCleanup deletes all consumedCardInstanceIds from deck/hand/discard/exhaust/lost/storage references.

Text convention:
Every consumable card text must end with:
Lost.

D. CONSUMABLE CATALOGUE
=======================

Consumables should be crafted as repeatable items.
Each crafted consumable item should create one consumable card instance in the run deck/loadout.
When the card is used, it goes to Lost and is deleted after the encounter, even if recovered and used again during the fight.

Balance:
- Standard consumables are cheap and made from basic materials.
- Quarry common consumables use quarry-specific common drops.
- Quarry rare consumables use rare/strange drops and should feel very strong.
- Consumables should be powerful enough to save for emergencies, but not so expensive that players never use them.

Lost pile edge cases:
- If a consumable is played, it must be marked consumed even if it later leaves Lost.
- If a Lost card is recovered to hand, it can be played again during that encounter.
- If the encounter ends and the card was ever marked consumed, delete it from all live piles and future deck/storage references.
- If an effect says return from discard but not Lost, it cannot return a Lost card.
- If an effect says return a consumable from Lost or discard, it can return from either.
